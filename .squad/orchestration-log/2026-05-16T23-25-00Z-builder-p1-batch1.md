## Outcomes

### P1.1 — Galaxy Data
**File:** `src/game/state/galaxy_data.gd`  
**Delivery:** Created GalaxyData singleton with inner classes `Planet` and `Lane`, 12 planets across 4 systems, 15 bidirectional lanes with demand profiles.  
**Decision:** D-P1.1-InnerClasses — Using inner classes for Planet/Lane keeps topology self-contained and eliminates file proliferation for pure data structures.  
**Status:** ✅ Committed

### P1.2 — Ship Catalog
**File:** `src/game/state/ship_catalog.gd`  
**Delivery:** Created ShipCatalog singleton with ShipInstance factory, 7 ship types (3 starter: SD, FX, RW; 4 unlock: XL, XT, HX, DR), capacity validation on instantiation, human-readable ID format `{type_id}-{counter}`.  
**Decision:** D-P1.2-ShipInstanceID — ID format `{type_id}-{counter}` (e.g., `sd-100-0001`) is human-readable and deterministic per session.  
**Status:** ✅ Committed

### P1.3 — Carrier Data
**File:** `src/game/state/carrier_data.gd`  
**Delivery:** Created CarrierData singleton with inner classes Route and ShipRef, 4 default carriers (Player, NPC-1, NPC-2, NPC-3) with 3000 cash each, persistent route registry.  
**Decision:** D-P1.3-ShipRef — ShipRef mirrors ShipInstance to avoid compile-time dependency during parallel P1.1–P1.3 work; P1.4 will unify.  
**Status:** ✅ Committed
